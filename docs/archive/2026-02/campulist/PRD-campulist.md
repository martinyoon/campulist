## 1. 프로젝트 개요

### 1.1 비전

> "대학 캠퍼스를 중심으로 한 한국형 크레이그리스트 — 대학생활의 모든 거래와 정보가 오가는 곳"

캠퍼스리스트는 크레이그리스트(Craigslist)의 **심플한 게시판 중심 구조**를 계승하면서, **한국 대학 캠퍼스 커뮤니티에 특화**된 분류형 마켓플레이스 플랫폼이다. 대학생, 교수, 교직원, 인근 상인이 하나의 플랫폼 안에서 중고 거래, 주거 탐색, 구인구직, 서비스 제공, 커뮤니티 활동을 할 수 있도록 한다.

### 1.2 문제 정의

| 문제 | 현재 상황 | 캠푸리스트 해법 |
|------|-----------|----------------|
| 대학생 중고거래 분산 | 에타 장터, 당근, 번개장터 등 분산 | 캠퍼스 단위 통합 마켓 |
| 대학가 주거 정보 부족 | 직방/다방은 대학가 단기·룸메이트 미지원 | 룸메이트·단기임대 전용 게시판 |
| 캠퍼스 구인구직 비효율 | 알바몬은 캠퍼스 특화 X, 과외는 개인 연결 | 과외·RA·TA·알바 통합 구인구직 |
| 대학가 상인 홍보 채널 부재 | 배달앱 수수료 부담, SNS 효과 한계 | 저비용 비즈니스 계정으로 직접 홍보 |
| 캠퍼스 서비스 연결 어려움 | 이사도움, 과외, 레슨 등 개별 탐색 | 서비스 게시판으로 통합 |

### 1.3 성공 지표 (KPI)

| 지표 | MVP 목표 (3개월) | 성장 목표 (12개월) |
|------|-------------------|---------------------|
| 등록 대학 수 | 3개 | 30개 |
| 월간 활성 사용자 (MAU) | 3,000 | 50,000 |
| 월간 게시글 수 | 5,000 | 100,000 |
| 비즈니스 계정 수 | 30 | 500 |
| 월 매출 | 100만원 | 3,000만원 |
| 게시글 → 거래 전환율 | 15% | 25% |

### 1.4 핵심 원칙

1. **심플 퍼스트**: 크레이그리스트처럼 게시판 중심의 단순한 구조 유지. 과도한 기능보다 빠른 게시·검색에 집중
2. **캠퍼스 신뢰**: 학교 이메일(.ac.kr) 인증으로 익명성과 신뢰성 균형
3. **무료 기본**: 개인 사용자의 기본 게시는 무료. 수익은 비즈니스 계정과 프리미엄 옵션에서
4. **모바일 퍼스트**: 대학생 90%+ 모바일 이용. 모바일 최적화 우선
5. **확장 가능한 구조**: 대학 단위로 독립 운영되면서 전국으로 확장 가능한 설계

**상세 명세 - F-01-02 학교 이메일 인증:**

```
[인증 플로우]
1. 사용자가 학교 이메일 주소 입력 (예: hong@snu.ac.kr)
2. 시스템이 .ac.kr 도메인 검증
3. 6자리 인증 코드 이메일 발송
4. 사용자가 코드 입력 → 인증 완료
5. 프로필에 "서울대학교 인증" 뱃지 표시
6. 해당 대학 게시판 쓰기 권한 부여

[예외 처리]
- 코드 만료: 10분
- 재발송: 최대 3회/시간
- 학교 도메인 DB에 없는 경우 → 수동 검토 요청
- 졸업생(만료 이메일) → 졸업생 인증 별도 프로세스
```

**상세 명세 - F-02-01 게시글 작성:**

```
[게시글 데이터 구조]
{
  id: string (UUID)
  title: string (2~50자)
  body: string (최대 5,000자)
  category: {
    major: string    // 대분류 (마켓/주거/일자리/커뮤니티/서비스/비즈니스)
    minor: string    // 소분류 (중고교재/전자기기/원룸 등)
  }
  price: number | null      // null = 가격 미정/무료
  priceNegotiable: boolean  // 가격 협의 가능 여부
  images: string[]          // 최대 10장
  tags: string[]            // 최대 5개 태그
  location: {
    university: string      // 소속 대학
    detail: string          // 상세 위치 (선택)
  }
  status: enum              // ACTIVE | RESERVED | COMPLETED | HIDDEN
  author: {
    id: string
    nickname: string
    verified: boolean       // 학교 인증 여부
    rating: number          // 거래 평점
  }
  viewCount: number
  likeCount: number
  createdAt: datetime
  updatedAt: datetime
  bumpedAt: datetime        // 끌어올리기 시각
}
```

### 5.3 핵심 사용자 플로우

#### 플로우 1: 중고 물품 판매

```
글쓰기 탭 → 카테고리 선택(마켓>전자기기) → 제목/가격/설명 입력
→ 사진 촬영/업로드 → 태그 추가 → [프리미엄 옵션 선택(선택)]
→ 등록 → 채팅 수신 대기 → 구매자와 협상 → 거래완료 처리
→ 후기 작성 요청 수신
```

#### 플로우 2: 룸메이트 찾기

```
주거 > 룸메이트 진입 → 필터 설정(학교, 성별, 예산)
→ 게시글 탐색 → 상세 확인 → 채팅 문의 → 만남/방문
→ 계약 합의 → 거래완료
```

#### 플로우 3: 상인 홍보

```
비즈니스 계정 가입 → 플랜 선택(프로) → 결제
→ 캠퍼스 비즈니스 > 맛집에 홍보글 작성 → 학생 할인 이벤트 등록
→ 대시보드에서 조회수/반응 확인 → 효과 분석
```
